Example Ruby Class:

class ClothingStore
def initialize(price_point, location)
@price_point = price_point
@location = location
end

def demographic
puts @price_point
puts @location
end
end

ClothingStore.new("high", "urban").demographic

Before discovering classes, we were focused on building methods. By defining a method, you're able to create a new set of commands that carry out a task built up of smaller tasks, adding up to create a larger concept. For example, if you wanted to create a method that addresses envelopes, the method would be comprised of smaller tasks adding up to the whole, such as gathering information, sorting it, alphabetizing it, ordering numerically, etc.

Classes take us to the next step. Now we're able to do the same with the methods themselves. Classes are a larger concept, creating a family with the innerworkings of the collaborating methods.

Instance methods are particularly crucial in this new complexity of interdependency. I like to think about them in relation to a long legal document, where the same variables such as "name," "social security number", or "net worth" appear repeatedly in different sections of the document. Instance methods remind me of "autofill", when the computer automatically fills in the corresponding entries throughout the document, after only a single entry.

This is set in place with the very first method used in the Class structure: "initialize." This is where the instance variables are defined so that they are cemented into the methods of the class in all instances.

They are the links that make the innerworkings (methods) of the class weave through one another to produce the desired output.
